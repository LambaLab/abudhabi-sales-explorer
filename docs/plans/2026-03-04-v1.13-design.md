# v1.13 Design — Profile header, Google avatar, realtime notification, ChatInput fix

Date: 2026-03-04

## Overview

Four user-reported issues from v1.12 screenshots:

1. Profile page still shows the full app header — user wants it hidden completely, with only a back `←` button (like X/Twitter)
2. Google OAuth profile photo not appearing — `profiles` table never populated with `avatar_url`
3. Real-time feed updates exist but auto-scroll; user wants a "↑ N new posts" badge instead
4. ChatInput floats in the middle of the profile page instead of sticking to the bottom

---

## Issue 1 — Remove header on profile routes

**Root cause:** `App.jsx` always renders `<header>`. v1.12 only swapped the center nav, leaving logo, version, theme toggle, ProfileMenu visible.

**Design:**
- In `App.jsx`: when `isProfile` is truthy, skip rendering `<header>` entirely.
- In `ProfilePage.jsx`: add a slim back `←` button at the top-left of the page content (sticky/fixed within the scroll area), matching X's profile UX. Button calls `navigate(-1)`.
- The App.jsx header and all its children (logo, version, theme toggle, ProfileMenu) disappear on profile routes.

---

## Issue 2 — Google profile photo not loading

**Root cause:** Supabase OAuth populates `auth.users.user_metadata.avatar_url` from Google, but the app's `profiles` table is never updated with this value. `useProfileFeed` reads from `profiles`, so it gets null/empty `avatar_url`.

**Design:**
- In `useAuth.js`: in `onAuthStateChange`, when `event === 'SIGNED_IN'`, upsert the `profiles` table with `id`, `display_name` (from `user_metadata.full_name`), and `avatar_url` (from `user_metadata.avatar_url`) using `onConflict: 'id'`.
- This ensures every sign-in syncs the Google photo to `profiles`, fixing it for all users.

---

## Issue 3 — "New posts" notification badge

**Root cause:** `useFeed.js` already has Supabase Realtime — new posts from other users DO arrive live. But `FeedPage.jsx` auto-scrolls to bottom on every `posts.length` increase.

**Design:**
- In `FeedPage.jsx`: add a `newPostCount` ref/state.
- In the `posts.length` useEffect: when `posts.length > prevCount.current`:
  - If the newest post's `userId === user?.id`: it's the current user's own post — auto-scroll as before.
  - Otherwise: increment `newPostCount`, do NOT auto-scroll.
- Show a "↑ N new posts" badge (styled like the existing "Jump to latest" button) above the ChatInput when `newPostCount > 0`.
- Clicking the badge: scroll to bottom, reset `newPostCount` to 0.
- When user manually scrolls to bottom (scroll event shows they're near the bottom), auto-reset `newPostCount`.

---

## Issue 4 — ChatInput positioning on profile page

**Root cause:** `ProfilePage.jsx` uses `<main className="relative flex-1 overflow-y-auto">` — `absolute bottom-0` children inside a scrollable container scroll with the content instead of sticking to the viewport bottom.

**Design:** Match `FeedPage.jsx`'s layout pattern exactly:
```
<div className="relative flex-1 flex flex-col min-h-0">   // outer, non-scrolling, is the anchor
  <main className="flex-1 overflow-y-auto">               // inner, scrolls
    <div>...profile content with pb-24...</div>
  </main>
  {isOwnProfile && !loading && (
    <div className="absolute bottom-0 ...">               // sticks to outer div's bottom
      <ChatInput />
    </div>
  )}
</div>
```
