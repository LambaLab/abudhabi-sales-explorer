# v1.15 — Reply UX & Avatar Design

**Date:** 2026-03-05
**Status:** Approved

---

## Overview

Four features improving the reply experience and user identity:

1. Auto-scroll to bottom after submitting a reply
2. New-reply notification pill for realtime replies from other users
3. Facebook-style reply layout (avatar left, name above bubble)
4. Profile picture upload via Supabase Storage

---

## Feature 1 — Auto-scroll after submit

**Problem:** After the user submits a reply they must manually scroll down to see the streaming typing indicator begin.

**Solution:** In `ReplyInput` (defined inside `PostCard.jsx`), after calling `onSubmit(postId, trimmed)` inside `handleSubmit`, immediately call:

```js
bottomRef.current?.scrollIntoView({ behavior: 'smooth' })
```

`ReplyInput` is a closure inside `PostCard`, so `bottomRef` is already in scope — no prop threading required.

**Result:** User hits Enter → reply container scrolls to bottom → streaming typing indicator is visible immediately.

---

## Feature 2 — New-reply notification pill

**Problem:** When a realtime reply from another user arrives, the current user may be scrolled up and miss it.

**Solution:** A pill appears at the bottom of the reply container when unread realtime replies exist.

### State (added to PostCard)

| State | Type | Purpose |
|---|---|---|
| `unseenCount` | `number` | Replies that arrived while user was scrolled up |
| `isAtBottom` | `boolean` | Whether the reply container is scrolled to the bottom |

### Logic

- Add `scrollRef` to the scrollable reply container `div`.
- Attach a `scroll` listener: `isAtBottom = scrollTop + clientHeight >= scrollHeight - 8` (8px threshold).
- In the `useEffect` that watches `post.replies`:
  - If length increased AND `!isAtBottom` AND the newest reply's `userId !== currentUser.id` → increment `unseenCount`.
- When user scrolls to bottom → reset `unseenCount = 0`.

### Pill UI

```jsx
{unseenCount > 0 && (
  <button
    onClick={() => { bottomRef.current?.scrollIntoView({ behavior: 'smooth' }); setUnseenCount(0) }}
    className="sticky bottom-2 mx-auto flex items-center gap-1.5 bg-accent text-white
               text-xs font-medium rounded-full px-3 py-1.5 shadow-md
               animate-slide-up"
  >
    ↓ {unseenCount} new {unseenCount === 1 ? 'reply' : 'replies'}
  </button>
)}
```

Position: rendered as the last child of the scrollable container, `sticky bottom-2`.

## Feature 3 — Facebook-style reply layout

**Problem:** `UserBubble` currently right-aligns the prompt bubble with a small avatar on the right and name below. Users want a Facebook/Instagram-style comment layout.

### Before

```
[prompt bubble]  [avatar]
       [name · timestamp]
```

### After

```
[avatar]  [name · timestamp]
          [prompt bubble]
```

### Implementation

**`UserBubble.jsx` restructure:**

```jsx
<div className="flex flex-row gap-3 items-start">
  {/* Left col — avatar */}
  <div className="shrink-0 h-8 w-8 rounded-full overflow-hidden ...">
    {showImg ? <img ... /> : <span>{initials(displayName)}</span>}
  </div>

  {/* Right col */}
  <div className="flex flex-col gap-1 min-w-0">
    {/* Name row */}
    <p className="flex items-center gap-1 text-xs text-slate-400">
      {userId && displayName && (
        <Link to={`/profile/${userId}`} className="font-medium text-accent hover:underline">
          {displayName}
        </Link>
      )}
      {(userId && displayName) && <span aria-hidden="true">·</span>}
      {relativeTime(createdAt)}
    </p>

    {/* Prompt bubble */}
    <div className="rounded-2xl bg-slate-100 dark:bg-slate-800 px-4 py-2.5 text-sm ...">
      {stripHint(prompt)}
    </div>
  </div>
</div>
```

Avatar size increases from `h-7 w-7` to `h-8 w-8` for better legibility.

## Feature 4 — Profile picture upload

### Supabase Storage setup

New migration file: `supabase/migrations/20260305_storage_avatars.sql`

```sql
-- Create public avatars bucket
INSERT INTO storage.buckets (id, name, public)
VALUES ('avatars', 'avatars', true)
ON CONFLICT (id) DO NOTHING;

-- Authenticated users can upload into their own folder
CREATE POLICY "Users can upload their own avatar"
ON storage.objects FOR INSERT TO authenticated
WITH CHECK (
  bucket_id = 'avatars'
  AND auth.uid()::text = (storage.foldername(name))[1]
);

-- Authenticated users can update their own objects
CREATE POLICY "Users can update their own avatar"
ON storage.objects FOR UPDATE TO authenticated
USING (
  bucket_id = 'avatars'
  AND auth.uid()::text = (storage.foldername(name))[1]
);

-- Public read
CREATE POLICY "Avatars are publicly readable"
ON storage.objects FOR SELECT TO public
USING (bucket_id = 'avatars');
```

### UI (ProfilePage.jsx)

- Only shown when viewing **own** profile (`profile.id === currentUser.id`).
- Camera icon badge on the bottom-right of the avatar circle (`absolute bottom-0 right-0`).
- Clicking the badge triggers a hidden `<input type="file" accept="image/*">`.
- While uploading: spinner overlay on avatar, button disabled.

### Upload flow

```js
async function handleAvatarUpload(file) {
  if (!file || file.size > 5 * 1024 * 1024) return // max 5 MB
  setUploading(true)
  const ext = file.name.split('.').pop()
  const path = `${userId}/${Date.now()}.${ext}`
  const { error: uploadError } = await supabase.storage
    .from('avatars').upload(path, file, { upsert: true })
  if (uploadError) { setUploading(false); return }
  const { data: { publicUrl } } = supabase.storage
    .from('avatars').getPublicUrl(path)
  await supabase.from('profiles').update({ avatar_url: publicUrl }).eq('id', userId)
  setProfile(p => ({ ...p, avatar_url: publicUrl }))
  setUploading(false)
}
```

### Bonus — Optimistic author data on pending replies

**Problem:** When `analyzeReply` creates the optimistic pending reply, `author` is undefined so the bubble shows initials while streaming.

**Fix:** When building the pending reply object in `analyzeReply` (or wherever the optimistic entry is created), attach the current user's profile:

```js
author: { display_name: currentUser.display_name, avatar_url: currentUser.avatar_url }
```

This makes the avatar + name appear immediately during streaming without waiting for the DB roundtrip.
