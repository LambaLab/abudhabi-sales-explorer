# v1.14 Design — Real-time replies + reply author name links

Date: 2026-03-05

## Overview

Two issues reported after v1.13 launch:

1. Replies posted by one user do not appear in real-time for other users — a page refresh is required.
2. Replies show the user's avatar but not their display name, and there is no link to the author's profile.

---

## Issue 1 — Real-time reply sync

**Root cause:** The client-side subscription in `useFeed.js` is already correct — the channel subscribes to `replies` INSERT events, performs a secondary fetch to get the full reply with author join, calls `fromDbReply`, and patches the matching post's replies array. The dedup check correctly ignores the echo of the current user's own reply while allowing other users' replies through.

The problem is **database-side**: Supabase Postgres Changes only publishes events for tables that are explicitly added to the `supabase_realtime` publication. The `posts` table is in it (proven by working post realtime), but `replies` is almost certainly not.

**Fix:** A one-line SQL migration:
```sql
ALTER PUBLICATION supabase_realtime ADD TABLE replies;
```

**Artifact:** `supabase/migrations/20260305_enable_realtime_replies.sql`

This file is the permanent record of the migration. It must also be applied manually in the Supabase Dashboard → SQL Editor (or via Supabase CLI `supabase db push`).

No client code changes are required for real-time reply sync — the subscription is already wired up correctly.

---

## Issue 2 — Reply author name + profile link

**Current state:** `UserBubble` renders a right-aligned accent bubble + avatar + timestamp. The author's `display_name` is not shown anywhere. There is no link to the author's profile.

**Data available:** `reply.userId` (camelCase, from `fromDbReply`) and `reply.author.display_name` (snake_case, passed through from DB join). Both are consistently available for all replies — those loaded on initial page fetch and those arriving via realtime.

**Design:**

Add a `userId` prop to `UserBubble`. Below the `[bubble + avatar]` row, show the author name and timestamp on the same line (right-aligned):

```
[  bubble text ............. ] [ avatar ]
                       Name · just now
```

- `display_name` rendered as `<Link to="/profile/{userId}">` — `text-xs font-medium text-accent hover:underline`
- Separated from timestamp by ` · `
- Entire bottom line: `flex justify-end items-center gap-1 text-xs text-slate-400`
- Only rendered when `userId` and `author?.display_name` are both truthy (graceful fallback — if missing, shows timestamp alone as before)

**Files changed:**

1. `src/components/ReplyCard.jsx` — pass `userId={reply.userId}` to `<UserBubble>`
2. `src/components/UserBubble.jsx` — add `userId` prop; add name+link to the timestamp row
3. `src/components/UserBubble.test.jsx` — add tests for name link presence, profile href, and missing-userId fallback
